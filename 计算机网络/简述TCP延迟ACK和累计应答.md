# 简述TCP延迟ACK和累计应答

## 延迟ACK
**延迟应答是指：TCP在接收到对端的报文后，并不会立刻发送ack，而是等待一段时间发送ack，以便将ack和要发送的数据一块发送。当然ack不能无限延长，否则对端会认为包超时而造成报文重传。**</br>
**Linux采用动态调节算法来确认延时时间**

## 累计应答
**累计应答是指：为了保证 <font color = red>顺序性</font>，每个包都有一个ID，在建立连接的时候，会商定起始ID是多少，然后按照ID一个一个发送，而为了保证不丢包，对应发送的包都要进行应答，但是不是一个个应答，而是会 <font color = red>应答某个之前的ID</font>，该模式被称为累计应答**
